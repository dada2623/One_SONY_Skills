# wishlist-manager

A focused wrapper around Notion API for your **Wishlist** database with intelligent product link parsing.

## Database IDs

- **Database ID**: `c1b6e15bc8e5472897f80fa3b0a18a02`
- **Data Source ID**: `e957af75-e069-4a68-be37-8e54b2eb2fb8`
- **Category Database ID**: `d8bc0e5b-4d7d-4271-835d-9b726055b9e0` (for 类别 relation)

## Setup

Ensure Notion API key is saved to `~/.config/notion/api_key` (Windows: `%USERPROFILE%\.config\notion\api_key`)

## Field Specifications

When creating or updating wishlist items, follow these field rules:

### 项目 - Required
- **Purpose**: Briefest possible description of the item
- **Format**: 2-4 words maximum, no brand, no model details
- **Examples**: "智能眼镜", "VR 头显", "手柄", "显示器", "机械键盘", "外套", "登山包"
- **Extraction**: Extract the core product category from product title

### 型号
- **Purpose**: Brand and specific model information
- **Format**: "品牌 型号" or "品牌 系列 规格"
- **Examples**:
  - "奥睿科 ORICO MiniLink-M48PR"
  - "海备思 Hagibis 胡桃木底座"
  - "理想 Livis 科技灰 哑光 定制蔡司镜片 Superb 自由环面"
  - "Valve Steam Frame"
  - "SUPERTOFU 龙虾扣 法工蓝 洗水做旧 工装夹克 AW25"

### 类别
- **Purpose**: Item category for filtering
- **Type**: Relation to another database
- **Handling**: This is a relation field. When creating, either:
  1. Leave empty and let user set manually in Notion
  2. Query the category database to find matching page ID
- **Common categories**: 影音设备, 专业工具, 消费电子, 配件, 家具收纳, 穿搭, 美妆, 书籍, 装饰, 运动健身, 旅行&旅游, 收纳箱包, 个卫洗护, 游戏, 礼物, 家用电器, 爱好

### 价格/预算
- **Purpose**: Current price or budget estimate
- **Format**: Integer number (no currency symbol)
- **Source**: Extract from link if available, otherwise ask user

### 平台
- **Purpose**: Purchase platform
- **Type**: Select
- **Options**: 京东, 淘宝, 闲鱼, 拼多多, 天猫, Apple 官网, 小米有品, 小米商城, 美团, 线下, 抖音, 二手
- **Auto-detection rules**:
  - `taobao.com`, `tb.cn`, `e.tb.cn` → 淘宝
  - `tmall.com` → 天猫
  - `jd.com`, `item.jd.com` → 京东
  - `pinduoduo.com`, `yangkeduo.com` → 拼多多
  - `2.taobao.com`, `idlefish` → 闲鱼
  - `apple.com.cn` → Apple 官网
  - `xiaomiyoupin.com` → 小米有品
  - `mi.com` → 小米商城
  - `douyin.com` → 抖音
  - If cannot detect, ask user

### URL
- **Purpose**: Product link
- **Format**: Desktop URL (convert mobile URLs to desktop)
- **Mobile to Desktop conversion**:
  - 淘宝 mobile: `h5.m.taobao.com/*` → desktop: `item.taobao.com/item.htm?id=xxx`
  - 淘宝 short: `e.tb.cn/*`, `m.tb.cn/*` → resolve redirect to get full URL
  - 京东 mobile: `m.jd.com/*` → desktop: `item.jd.com/xxx.html`
  - Keep the original link if conversion fails

### 状态
- **Purpose**: Purchase status
- **Type**: Status
- **Options**: 未买, 已拔草, 购入竞品, 退货/出二手, 完成
- **Default**: 未买 (unless user explicitly states otherwise)

### 优先级
- **Purpose**: Purchase priority
- **Type**: Select
- **Options**: P0, P1, P2, P3 (P0 = highest)
- **Default**: P0

### 完成日期
- **Purpose**: Date when item was purchased/completed
- **Type**: Date
- **Format**: ISO 8601 date string (YYYY-MM-DD)
- **Usage**: Set automatically when status changes to "完成" or manually by user

### 创建时间
- **Purpose**: When the item was added to wishlist
- **Type**: Created time (auto-generated by Notion)
- **Read-only**: Automatically set when page is created

## Updating Wishlist Items

When user wants to update an item (e.g., mark as purchased, change priority):

### Update Status to "完成"
When status changes to "完成", automatically set 完成日期 to today's date:

```javascript
const updateData = {
  properties: {
    "状态": { status: { name: "完成" } },
    "完成日期": { date: { start: "2026-03-23" } }
  }
};
PATCH https://api.notion.com/v1/pages/{page_id}
```

### Update Priority
```javascript
const updateData = {
  properties: {
    "优先级": { select: { name: "P1" } }
  }
};
```

### Update Price
```javascript
const updateData = {
  properties: {
    "价格/预算": { number: 299 }
  }
};
```

## Creating Wishlist Items

When user provides a product link or description, follow this workflow:

### Step 1: Parse Input
Extract information from user input:
1. **URL**: Extract any links present
2. **Product title**: Extract from link text or user description
3. **Platform**: Auto-detect from URL domain
4. **Price**: Try to extract from link/description, otherwise ask

### Step 2: Generate Field Values
Based on extracted info:
1. **项目**: Simplify product title to 2-4 word category
2. **型号**: Combine brand + model from full product title
3. **平台**: Use detected platform
4. **URL**: Convert mobile URL to desktop URL
5. **状态**: Default to "未买"
6. **优先级**: Default to "P0"
7. **价格/预算**: Extract or ask user
8. **类别**: Leave empty (relation field, set manually or query category DB)

### Step 3: Create Page via Notion API

Use Node.js to call Notion API directly:

```javascript
const NOTION_API_KEY = require('fs').readFileSync(`${process.env.USERPROFILE}/.config/notion/api_key`, 'utf8').trim();
const DATABASE_ID = 'c1b6e15bc8e5472897f80fa3b0a18a02';

const pageData = {
  parent: { database_id: DATABASE_ID },
  properties: {
    "项目": { title: [{ text: { content: "工装夹克" } }] },
    "型号": { rich_text: [{ text: { content: "SUPERTOFU 龙虾扣 法工蓝 洗水做旧 AW25" } }] },
    "价格/预算": { number: 299 },
    "平台": { select: { name: "淘宝" } },
    "URL": { url: "https://item.taobao.com/item.htm?id=xxx" },
    "状态": { status: { name: "未买" } },
    "优先级": { select: { name: "P0" } }
  }
};

// POST to https://api.notion.com/v1/pages
```

### Step 4: Confirm with User
Show the created item details and ask if any fields need adjustment.

## API Reference

### Notion API Headers
```javascript
{
  'Authorization': `Bearer ${NOTION_API_KEY}`,
  'Notion-Version': '2025-09-03',
  'Content-Type': 'application/json'
}
```

### Key Endpoints
- Create page: `POST /v1/pages`
- Update page: `PATCH /v1/pages/{page_id}`
- Get page: `GET /v1/pages/{page_id}`
- Query database: `POST /v1/data_sources/{data_source_id}/query`
- Get database schema: `GET /v1/data_sources/{data_source_id}`

## Scripts

The `scripts/` directory contains bash scripts for Unix systems with `jq` installed. On Windows without `jq`, use Node.js directly as shown above.

- `scripts/create-page.sh` - Create a new page
- `scripts/list-pages.sh` - List all pages
- `scripts/get-page.sh` - Get page details
- `scripts/update-page.sh` - Update page properties
- `scripts/delete-page.sh` - Archive a page
- `scripts/add-block.sh` - Add content block to page

## Node.js Helper Script

A Node.js script `wishlist.js` is provided for Windows environments:

```bash
# List pages
node wishlist.js list

# Create page from product link
node wishlist.js create "【淘宝】...「产品名称」 https://..."

# Update page
node wishlist.js update <page_id> '{"状态":"已买"}'
```
